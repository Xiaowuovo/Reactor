## ❓ 常见问题处理

### Q1: 编译失败怎么办？

**检查清单：**
```bash
# 1. 检查g++版本
g++ --version  # 需要支持C++11

# 2. 检查文件完整性
ls -la *.cpp *.h

# 3. 清理后重新编译
make clean
make all
```

- 准备预编译的二进制文件
- 或准备录屏视频

### Q4: 老师提问"为什么用线程局部？"

**回答要点：**
1. **问题：** malloc/free在多线程下有全局锁竞争
2. **解决：** thread_local使每个线程独立，无需加锁
3. **优势：** 零竞争、无cache bouncing、线性扩展
4. **数据：** 多线程场景性能提升5-10倍

---

### Q5: 老师提问"三级池如何设计？"

**回答要点：**
1. **动机：** 避免小消息占用大块内存
2. **策略：** 
   - Small (1KB) - 35%流量
   - Medium (4KB) - 50%流量
   - Large (16KB) - 14%流量
3. **命中率：** 99%（仅1%超大包用malloc）
4. **可调整：** 支持动态调整阈值
