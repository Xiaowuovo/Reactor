## 核心设计模式

### 1. Reactor 模式 (主从Reactor)

```
             新连接请求
                 │
                 ▼
        ┌────────────────┐
        │  Main Reactor  │  accept() 新连接
        │   (Acceptor)   │
        └────────┬───────┘
                 │ 分发连接（Round-Robin）
        ┌────────┼────────┬────────┐
        │        │        │        │
        ▼        ▼        ▼        ▼
    ┌───────┐┌───────┐┌───────┐┌───────┐
    │  Sub  ││  Sub  ││  Sub  ││  Sub  │
    │Reactor││Reactor││Reactor││Reactor│
    │  #1   ││  #2   ││  #3   ││  #4   │
    └───┬───┘└───┬───┘└───┬───┘└───┬───┘
        │        │        │        │
        └────────┴────────┴────────┘
                 │
                 ▼ 投递耗时任务
        ┌────────────────┐
        │  Worker Pool   │
        └────────────────┘
```

**优势：**
- 主线程专注于接收新连接
- 从线程处理已有连接的I/O
- 负载均衡，充分利用多核

### 2. One Loop Per Thread 模型

```
Thread 1 (Main):              Thread 2 (IO):              Thread 3 (IO):
┌──────────────┐              ┌──────────────┐            ┌──────────────┐
│  EventLoop   │              │  EventLoop   │            │  EventLoop   │
│  ┌────────┐  │              │  ┌────────┐  │            │  ┌────────┐  │
│  │ Epoll  │  │              │  │ Epoll  │  │            │  │ Epoll  │  │
│  └───┬────┘  │              │  └───┬────┘  │            │  └───┬────┘  │
│      │       │              │      │       │            │      │       │
│  ┌───▼────┐  │              │  ┌───▼────┐  │            │  ┌───▼────┐  │
│  │Acceptor│  │              │  │ Conn1  │  │            │  │ Conn3  │  │
│  └────────┘  │              │  │ Conn2  │  │            │  │ Conn4  │  │
│              │              │  └────────┘  │            │  └────────┘  │
│ thread_local │              │ thread_local │            │ thread_local │
│ MemoryPool   │              │ MemoryPool   │            │ MemoryPool   │
└──────────────┘              └──────────────┘            └──────────────┘
```

**特点：**
- 每个线程独立的事件循环
- 线程局部数据，无需加锁
- 避免跨线程数据访问

## 模块详细设计

### EventLoop（事件循环）

**职责：**
```
┌─────────────────────────────────────┐
│         EventLoop核心功能            │
├─────────────────────────────────────┤
│ 1. 事件监听  → epoll_wait()         │
│ 2. 事件分发  → handleEvent()        │
│ 3. 定时器    → timerfd              │
│ 4. 跨线程通信→ eventfd + 任务队列   │
│ 5. 连接管理  → map<fd, Connection>  │
└─────────────────────────────────────┘
```

**事件循环流程：**
```cpp
while (!stop_) {
    // 1. 等待事件（可设置超时）
    vector<Channel*> active_channels = epoll_->loop(timeout);
    
    // 2. 处理就绪事件
    for (Channel* ch : active_channels) {
        ch->handleEvent();  // 读/写/错误回调
    }
    
    // 3. 执行任务队列（跨线程投递的任务）
    executePendingTasks();
    
    // 4. 检查超时连接
    checkTimeouts();
}
```

### Connection（连接管理）

**状态机：**
```
    [新连接]
       │
       ▼
  ┌─────────┐   读事件    ┌──────────┐
  │Connected├────────────►│ Reading  │
  └────┬────┘             └─────┬────┘
       │                        │
       │ 写事件                  │ 数据就绪
       ▼                        ▼
  ┌─────────┐   发送完成  ┌──────────┐
  │ Writing │◄────────────┤Processing│
  └────┬────┘             └──────────┘
       │
       │ 关闭/错误
       ▼
  ┌─────────┐
  │ Closed  │
  └─────────┘
```

**缓冲区设计：**
```
Connection {
    Buffer inputBuffer_;   // 接收缓冲区
    Buffer outputBuffer_;  // 发送缓冲区
}

Buffer内部结构：
┌─────────────────────────────────────┐
│  [4字节长度][消息内容][4字节长度]... │
│   ↑ 网络字节序                       │
└─────────────────────────────────────┘

处理粘包/拆包：
- 读取4字节长度
- 检查是否完整
- 完整则解析，不完整则缓存
```

## 数据流转图

### 新连接建立流程

```
客户端                   服务器
  │                        │
  │ ─── SYN ────────────► │ Acceptor监听
  │ ◄── SYN+ACK ──────── │
  │ ─── ACK ────────────► │
  │                        │ accept() 返回clientfd
  │                        ▼
  │                  TcpServer::newConnection()
  │                        │
  │                        ├─► 选择SubLoop (fd % num_threads)
  │                        ├─► 创建Connection对象
  │                        ├─► 设置回调函数
  │                        ├─► 添加到EventLoop
  │                        └─► 开始监听读事件
  │                        │
  │ ◄─── 可以通信 ───────► │
```

### 数据接收流程

```
┌──────────┐
│  Epoll   │ EPOLLIN事件触发
└─────┬────┘
      │
      ▼
┌──────────────────────┐
│ Connection::onRead() │
├──────────────────────┤
│ 1. read() 循环读取   │
│    直到EAGAIN        │
│ 2. 追加到inputBuffer │
│ 3. pickmessage()拆包 │
└──────┬───────────────┘
       │
       ▼
┌────────────────────────────┐
│ TcpServer::onMessage()     │
│ (用户业务逻辑回调)          │
├────────────────────────────┤
│ 1. 处理消息内容            │
│ 2. 生成回复                │
│ 3. Connection::send()发送   │
└────────────────────────────┘
```

### 数据发送流程

```
应用层调用 Connection::send(data)
       │
       ▼
追加到 outputBuffer_
       │
       ▼
enableWriting() 注册写事件
       │
       ▼
EPOLLOUT 事件触发
       │
       ▼
Connection::onWrite()
       │
       ├─► send() 发送数据
       ├─► 检查是否发送完成
       └─► 完成则 disableWriting()
```

### 内存池内部结构

```
MemoryPool (单级池):

预分配内存块:
┌───────────────────────────────────────────────────────────┐
│ Chunk 1 (malloc 一大块)                                   │
├─────────┬─────────┬─────────┬─────────┬─────────┬────────┤
│ Block 1 │ Block 2 │ Block 3 │ Block 4 │ Block 5 │  ...   │
│  1024B  │  1024B  │  1024B  │  1024B  │  1024B  │        │
└────┬────┴────┬────┴────┬────┴────┬────┴────┬────┴────────┘
     │         │         │         │         │
     ▼         ▼         ▼         ▼         ▼
   ┌───┐     ┌───┐     ┌───┐     ┌───┐     NULL
   │ ●─┼────►│ ●─┼────►│ ●─┼────►│ ●─┼────►
   └───┘     └───┘     └───┘     └───┘
    ▲
    │
free_list_ (空闲链表头)

分配操作 (O(1)):
  1. block = free_list_
  2. free_list_ = block->next
  3. return block

释放操作 (O(1)):
  1. block->next = free_list_
  2. free_list_ = block
```

### 三级池智能分配策略

```
allocate(size):
    │
    ├─ size <= 1024   ──► Small Pool  (1KB块)
    │                      └─► 用于：控制命令、心跳包
    │
    ├─ size <= 4096   ──► Medium Pool (4KB块)
    │                      └─► 用于：普通消息、JSON数据
    │
    ├─ size <= 16384  ──► Large Pool  (16KB块)
    │                      └─► 用于：大文件传输、视频流
    │
    └─ size > 16384   ──► malloc (系统分配)

```

## 性能优化关键点

### 1. 内存对齐优化

```cpp
// 64字节缓存行对齐
MemoryPool::MemoryPool(size_t block_size, 
                       size_t initial_blocks, 
                       size_t alignment = 64)

对齐后的好处:
┌─────────────────────────────────────┐
│  CPU Cache Line (64 bytes)          │
│  ┌──────────────────────────────┐   │
│  │  完整的内存块                 │   │
│  │  (无跨缓存行访问)             │   │
│  └──────────────────────────────┘   │
└─────────────────────────────────────┘

未对齐的问题:
┌─────────────────────┬───────────────┐
│ Cache Line 1        │ Cache Line 2  │
│  ┌────────────┬─────┼─────┐         │
│  │  Block  一半│     │另一半│         │
│  └────────────┴─────┼─────┘         │
└─────────────────────┴───────────────┘
      ↑ Cache Miss! 需要2次内存访问
```

### 2. 边缘触发 + 非阻塞I/O

```cpp
// EPOLLET 边缘触发
channel->useet();

优势:
- 事件只触发一次，减少epoll_wait调用
- 需要循环读取直到EAGAIN

读取示例:
while (true) {
    ssize_t n = read(fd, buffer, sizeof(buffer));
    if (n > 0) {
        // 处理数据
    } else if (n == 0) {
        // 对端关闭
        break;
    } else {
        if (errno == EAGAIN) {
            // 数据读完，退出循环
            break;
        } else if (errno == EINTR) {
            // 信号中断，继续读
            continue;
        } else {
            // 错误处理
            break;
        }
    }
}
```

### 3. 零拷贝技术潜力

```
当前实现:
  用户空间 → 内核空间 → 网卡

优化方向 (sendfile/splice):
  用户空间 ────────────► 网卡
           (零拷贝)
```

## 总结

### 架构优势

1. **高性能**
   - 线程局部内存池：5-10x性能提升
   - Reactor模式：高并发处理能力
   - 边缘触发：减少系统调用

2. **高扩展性**
   - One Loop Per Thread：线性扩展
   - 无锁设计：无竞争瓶颈
   - 模块化设计：易于扩展

3. **易用性**
   - 清晰的回调接口
   - RAII资源管理
   - 完善的错误处理

4. **可靠性**
   - 超时管理
   - 优雅关闭
   - 智能指针管理生命周期

### 适用场景

- ✅ 高并发网络服务器
- ✅ 实时通信系统
- ✅ 游戏服务器
- ✅ RPC框架
- ✅ 消息中间件
