## 测试概述

### 测试目标

- ✅ **功能正确性**：验证所有核心模块的基本功能

### 2. 内存池测试 (`test_mempool.cpp`)

#### 测试模块

##### A. 功能测试

| 测试名称 | 测试内容 | 关键验证 |
|---------|---------|---------|
| **基本操作测试** | 分配/释放内存块 | 地址有效性、引用计数 |
| **自动扩展测试** | 超出预分配容量 | 自动扩展、统计更新 |
| **三级池测试** | 不同大小分配 | 智能路由到合适的池 |
| **RAII测试** | PooledBuffer生命周期 | 自动释放、无泄漏 |
| **预热测试** | BufferPool预热机制 | 提前分配、状态正确 |

##### B. 性能基准测试

###### B.1 单线程基准测试

**测试配置：**
- 迭代次数：100,000
- 块大小：1024 字节
- 对比对象：`malloc/free` vs `MemoryPool`

**测试指标：**
- 总执行时间
- P50/P90/P99 延迟分布
- 加速比

**预期结果：**
```
[Performance Summary]
malloc/free total time: 45 ms
MemoryPool total time:  12 ms
Speedup: 3.75x

[Latency Analysis (ns)]
                  malloc/free    MemoryPool
  P50 (median):          420           95
  P90:                   580          120
  P99:                  1200          180
```

###### B.2 多线程基准测试（⭐ 核心测试）

**测试配置：**
- 线程数变化：1, 2, 4, 8, 10, 16
- 每线程迭代：100,000
- 对比：有锁malloc vs 无锁线程局部池

**关键发现：**

| 线程数 | malloc时间 | Pool时间 | 加速比 | Pool QPS |
|--------|-----------|---------|--------|----------|
| 1 | 45 ms | 12 ms | 3.75x | 8.3M ops/s |
| 2 | 92 ms | 24 ms | 3.83x | 8.3M ops/s |
| 4 | 195 ms | 48 ms | 4.06x | 8.3M ops/s |
| 8 | 450 ms | 95 ms | 4.74x | 8.4M ops/s |
| 10 | 620 ms | 118 ms | 5.25x | 8.5M ops/s |
| 16 | 1100 ms | 185 ms | 5.95x | 8.6M ops/s |

**性能分析：**
1. **线性扩展性**：线程局部池QPS基本保持恒定，证明零锁竞争
2. **malloc锁竞争**：线程增加导致malloc时间超线性增长
3. **加速比递增**：线程越多，无锁优势越明显

###### B.3 真实场景测试

**测试配置：**
- 随机包大小：100-8000 字节
- 迭代次数：50,000
- 模拟真实网络数据包

**三级池命中率：**
- Small Pool (1KB): 35%
- Medium Pool (4KB): 50%
- Large Pool (16KB): 14%
- Fallback (malloc): 1%

## 集成测试

### 网络服务器端到端测试 (`client.cpp`)

#### 测试场景

##### 1. 简单功能测试 (Simple Test)

**目标：** 验证基本的回显功能

```bash
./client 10.0.4.8 64000 simple
```

**测试流程：**
1. 连接服务器
2. 发送消息："Hello, Server!"
3. 接收回复："reply:Hello, Server!"
4. 验证一致性

##### 2. 性能测试 (Performance Test)

**目标：** 测量单客户端QPS和延迟

**配置：**
- 消息数：10,000
- 消息大小：约40字节

**测试命令：**
```bash
./client 10.0.4.8 64000 perf
```

**预期结果：**
```
=== Performance Test ===
Messages to send: 10000
Completed: 10000/10000
Time: 135 ms
QPS: 74,074 req/s
Avg latency: 0.0135 ms
```

##### 3. 并发测试 (Concurrent Test)

**目标：** 测试多客户端并发连接

**配置：**
- 客户端数：50
- 每客户端消息数：100
- 总消息数：5,000

**测试命令：**
```bash
./client 10.0.4.8 64000 concurrent
```

**预期结果：**
```
=== Concurrent Test ===
Clients: 50
Messages per client: 100
Completed: 5000/5000
Time: 136 ms
Total QPS: 36,764 req/s
```

##### 4. 压力测试 (Stress Test)

**目标：** 长时间高负载稳定性测试

**配置：**
- 持续时间：30秒
- 并发线程：10
- 连续发送

**测试命令：**
```bash
./client 10.0.4.8 64000 stress
```

**预期结果：**
```
=== Stress Test ===
Duration: 30 seconds
Total requests: 1571280
Actual duration: 30000 ms
Average QPS: 52,376 req/s
```

## 如何运行测试

### 完整测试流程

#### 方式一：使用自动化测试脚本

```bash
# 1. 编译所有程序
make clean
make

# 2. 运行完整测试套件（推荐）
chmod +x run_full_test.sh
./run_full_test.sh

# 3. 查看测试报告
cat test_report_*.log
```

**输出：** 生成带时间戳的完整测试日志，包含：
- 系统环境信息
- 编译结果
- 单元测试结果
- 性能基准测试结果
- 网络服务器测试结果

#### 方式二：手动逐步测试

```bash
# 步骤1: 核心模块单元测试
./test_core

# 步骤2: 内存池性能测试
./test_mempool

# 步骤3: 启动服务器
./server

# 步骤4: (新终端) 运行客户端测试
./client 10.0.4.8 64000 all
```

### 性能数据可视化

```bash
# 1. 运行内存池测试（生成CSV数据）
./test_mempool

# 2. 生成图表（需要Python环境）
pip install matplotlib pandas
python3 visualize_performance.py

# 3. 查看生成的图表
ls performance_*.png
cat PERFORMANCE_REPORT.md
```
