## 核心技术亮点

### 1. Reactor 模式实现

**主从 Reactor 架构**：
- 1个主EventLoop负责accept
- N个从EventLoop负责I/O（可配置）
- M个工作线程处理业务（可配置）

**关键特性**：
- ✅ One Loop Per Thread
- ✅ 跨线程任务投递（eventfd）
- ✅ 边缘触发（EPOLLET）
- ✅ 非阻塞I/O
- ✅ 定时器管理（timerfd）

### 2. 内存池设计

**三级内存池架构**：
```
Small  (1KB)  ──→ 小消息、控制命令
Medium (4KB)  ──→ 普通业务数据  
Large  (16KB) ──→ 大文件、视频流
```

**性能提升**：
- 固定大小场景：**4.7x**
- 随机大小场景：**3.5x**
- 网络服务场景：**23-34%**

**内存优化**：
- 峰值内存减少：**36%**
- 内存碎片显著降低
- 长期运行稳定

### 3. 线程安全保障

**同步机制**：
- 互斥锁保护共享数据
- eventfd实现高效唤醒
- 原子操作保证标志位

**生命周期管理**：
- `std::shared_ptr` 自动引用计数
- `enable_shared_from_this` 安全获取this
- RAII管理资源释放

## 技术难点与解决方案

### 难点1：线程间通信

**问题**：工作线程处理完业务需要通知IO线程发送数据

**解决**：
```cpp
// eventfd + 任务队列
void EventLoop::queueinloop(std::function<void()> fn) {
    {
        std::lock_guard<std::mutex> lock(task_mtx_);
        taskqueue_.push(fn);
    }
    wakeup();  // 写eventfd唤醒IO线程
}
```

### 难点2：内存池的线程安全

**问题**：多线程同时分配/释放内存块

**解决**：
```cpp
void* MemoryPool::allocate() {
    std::lock_guard<std::mutex> lock(mutex_);  // 加锁保护
    // ... 分配逻辑
}
```

**优化方向**：per-thread pool（无锁）

### 难点3：连接超时管理

**问题**：如何高效检测并清理超时连接

**解决**：
```cpp
// 使用timerfd定时触发
void EventLoop::handletimer() {
    time_t now = time(0);
    for (auto it = conns_.begin(); it != conns_.end(); ) {
        if (it->second->timeout(now, timeout_)) {
            // 删除超时连接
            it = conns_.erase(it);
        } else {
            ++it;
        }
    }
}
```

### 难点4：粘包/拆包处理

**问题**：TCP流式协议需要正确拆分消息

**解决**：
```cpp
// 四字节报头协议
bool Buffer::pickmessage(std::string &ss) {
    int len;
    memcpy(&len, buf_.data(), 4);  // 读取长度
    
    if (buf_.size() < len + 4) return false;  // 不完整
    
    ss = buf_.substr(4, len);  // 提取消息
    buf_.erase(0, len + 4);    // 删除已处理
    return true;
}
```

## 答辩要点速查

### 核心问题回答模板

**Q1: 为什么选择Reactor模式？**

A: Reactor模式是高性能网络服务器的经典模式，优势：
1. 事件驱动，充分利用CPU
2. 非阻塞I/O，支持高并发
3. 代码结构清晰，易于维护
4. 被广泛应用（nginx、redis、muduo等）

**Q2: 内存池如何提升性能？**

A: 主要通过三个方面：
1. **减少系统调用**：预分配避免频繁malloc
2. **降低碎片**：固定大小块，无外部碎片
3. **提高局部性**：连续内存，缓存友好

测试数据显示**4.7倍**性能提升。

**Q3: 如何保证线程安全？**

A: 采用多层保护：
1. **互斥锁**保护共享数据（conns_、taskqueue_）
2. **eventfd**实现线程间高效通信
3. **智能指针**管理对象生命周期
4. **原子操作**保护标志位（stop_、disconnect_）

**Q4: 项目的创新点是什么？**

A: 三个主要创新：
1. **三级内存池**：适应不同大小网络包
2. **完整测试体系**：功能+性能+对比测试
3. **详细文档**：设计+性能+使用全覆盖

**Q5: 遇到的最大困难？**

A: 线程间通信的设计。最初使用条件变量，但唤醒效率低。后改用eventfd配合epoll，实现了高效唤醒，性能提升明显。
